# Davomat Bot

Telegram-бот системы давомата. Node.js + [telegraf](https://telegraf.js.org/).

Команды:
- `/today` — давомат за сегодня (итог + опоздавшие/отсутствующие)
- `/late` — кто опоздал сегодня
- `/month` — итоги за текущий месяц + топ опаздывающих
- `/start`, `/help`, `/ping`, `/health`

Бот читает данные через API давомата под учёткой администратора
(`ADMIN_EMAIL` / `ADMIN_PASSWORD` в `.env`).

## Запуск
```bash
cd DavomatBot
npm install
cp .env.example .env     # затем впишите BOT_TOKEN от @BotFather
npm run dev              # автоперезапуск (node --watch)
# или: npm start
```

## Переменные окружения (.env)
- `BOT_TOKEN` — токен бота от @BotFather (обязательно).
- `API_BASE_URL` — адрес API давомата (по умолчанию `https://localhost:7052/api`).
- `ADMIN_EMAIL` / `ADMIN_PASSWORD` — учётка администратора, под которой бот читает данные.
- `API_INSECURE_TLS` — `true`, чтобы принимать самоподписанный сертификат локального бэкенда (только для разработки).
- `REPORT_CHAT_ID` — id чата для ежедневного авто-отчёта (пусто = выключено).
- `REPORT_CRON` — расписание (cron), по умолчанию `30 18 * * *` (каждый день 18:30).
- `REPORT_TZ` — часовой пояс расписания (по умолчанию `Asia/Tashkent`).

## Авто-отчёт по расписанию
Если задан `REPORT_CHAT_ID`, бот сам отправляет отчёт за день в этот чат по `REPORT_CRON`.
Узнать id чата можно у [@userinfobot](https://t.me/userinfobot) (для группы — добавьте бота в группу).

## Структура
- `src/bot.js` — точка входа, команды бота.
- `src/api.js` — HTTP-клиент к API давомата.
